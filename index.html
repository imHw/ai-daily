Cursor × SpaceX：应用层 AI 的第一个「大满贯」模板

本周 builder 圈最热的话题，是 Cursor 与 SpaceX 的合作。Box CEO Aaron Levie 给了它一个分量很重的定位——这是 AI 应用层第一个规模化的成功案例：它跑通了「深耕垂直领域 + 充当模型路由器 + 应用层 GTM 与分发」这套打法，证明了在前沿模型与自训模型之间何时取舍、如何把市场机会真正吃下来。「这是第一个可规模化复制的执行范本。」

Cofounders 之一 Madhu Guru 则点出交易真正的奖品：那套 agentic harness——规划、上下文管理、工具调用、迭代、验证、记忆、错误恢复。它能成为大规模自动化知识工作的内核，任何产品体验都可以借它被重做成 AI 原生形态。Cursor 的稀缺之处在于，模型、评测、harness、应用层这一整条栈，它每一环都做得不差。

Anysphere 设计师 Ryo Lu 的调侃则更具想象力：「疯狂的想法——如果 X 和 Cursor 本来就是同一个东西呢？」一句话点破了「编辑器即 agent 平台」的野心。

大规模模拟人类：Simile 创始人 Joon Sung Park

YouTube 访谈 · 6 月 16 日

故事要从 2023 年斯坦福的「Smallville」说起——一个由 25 个生成式 agent 组成的虚拟小镇。每个 agent 有自己的人设，会早起、做日常、上班、建立关系，甚至自发组织起派对这类涌现行为。

Park 的核心洞察是：大语言模型已经把海量人类微观行为编码进了训练数据，只要「以正确的角度切入」，给定一个具体情境，模型就能生成可信的反应。这正是构建复杂 agent 社会一直缺的那味原料。

如今他创办 Simile，要把「模拟人类行为与社会」做成一个应用型 AI 实验室。他相信：科幻里足够成熟的文明都有两根支柱——某种 AGI，和某种用于指导社会的模拟系统。「五年前我不会这么说，但今天，我们第一次有机会真正动手建『模拟』这根柱子。」

用 Claude 接入 Apple Foundation Models 框架

claude.com/blog

Anthropic 发布了一个 Swift 包，让 Apple 开发者能通过 Apple 的 Foundation Models 框架直接调用 Claude。端侧小模型负责摘要、抽取这类快速本地任务；一旦遇到需要多步推理、代码生成的请求，就无缝交接给 Claude，还能联网搜索实时信息、执行代码做数据分析。

妙处在于：借助框架里 @Generable 注解返回的类型化 Swift 值，请求到达 Claude API 时已经是干净的结构化输入，而非原始用户文本。对用户是一段连续体验，背后每一步都用上了对的模型。

应用层的崛起：AI 工具为何没让开发者消失

GitHub 每月收到 1700 万个 Pull Request，数字还在指数级增长。但 COO Kyle Daigle 观察到的不是开发者被取代，而是使用代码的人群边界正在模糊——艺术学院学生、非科班出身者开始写代码，GitHub 对「开发者」的定义从未如此宽泛。与此同时，Box CEO Aaron Levie 的长推文在圈内引发共鸣：应用层并非薄薄一层 LLM 包装，驱动企业级智能体落地的复杂度，正在为这一层创造护城河。

Levie 总结了应用层的制胜要素：定制化工作流界面、模型路由能力、领域专家式的实施团队，以及懂行业语言的 GTM 动作。这些能力不会被「模型智能」一键抹平——企业需要的是今天就能用的变革方案，而非等待 AGI 的理论极限。与此呼应的是 Vercel CEO Guillermo Rauch 提出的类比：React 需要 Next.js 才能真正造出应用，AI SDK 也需要一个「部署智能体的实用方案」。开源模型 GLM 5.2 在 Next.js 评测中超过 Opus 4.8，模型竞争白热化的当下，抽象层和工具链的价值反而凸显。

另一边，YC 掌门人 Garry Tan 算了笔账：Fable 5 被封禁后，按 500 万日活开发者、17.8% 的工作量和 15% 的生产力提升计算，每小时损失 1200 万美元产出。这不只是模型可用性问题，更是整个生态对前沿工具的依赖度已深入骨髓。与之对应的冷静声音来自 Zara Zhang：在形成自己的品味和声音之前，别用 AI 写作——你甚至认不出它生成的是垃圾。技术的边界扩张与人的判断力培养，正在成为这一轮浪潮中同样重要的两极。

GitHub COO：为什么 AI 没有取代开发者

Every · 2025年6月

Every 主持人 Mike Taylor 对话 GitHub COO Kyle Daigle，探讨编码智能体前线的真实状况。GitHub 每月 1700 万 Pull Request 的指数增长，让他们站在新经济的最前沿。对话覆盖了用户画像变化、开源维护者的挑战，以及所有人都关心的话题：如何避免每月 200 美元的订阅膨胀成 2000 美元的用量账单。

节目还有个有趣细节：Mike 告诉 Kyle 自己用 AI 克隆了他来练习采访，Kyle 的回应出人意料。GitHub 对开发者的定义一直很宽泛——Kyle 自己就是艺术学院出身，在自称开发者之前就开始写代码了。这种包容性视角，正是 GitHub 面对 AI 时代用户群体剧变的底气所在。

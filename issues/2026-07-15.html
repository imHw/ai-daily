今日主线

多模型编排成新常态；开放权重逆袭网关流量；Artifacts 进化为创意画布

Builder 圈今天最热的话题是「多模型编排」。Swyx 公开了他的「大男孩项目」模型栈：用 Sol Ultra 做规划、Fable 5 做审查、Sonnet 5 / Terra Ultra / SWE 1.7 做实际编码——这标志着单一模型已无法满足专业开发者的需求，「哪个模型最好」的讨论正在被「如何组合模型」取代。同一时间，Thibault Sottiaux 暗示 ChatGPT 即将突破 800 万日活用户，ChatGPT Work 功能也在持续迭代。

开放权重阵营迎来里程碑数据。Guillermo Rauch 分享 Vercel AI Gateway 的最新统计：开放权重模型在网关 token 流量中的占比从 4 月的 11% 飙升至 29%。他同时介绍了 Vercel 的实验平台——让 Agent 自主设置和调优功能实验，打造「自优化网站」。这组数据印证了一个趋势：企业级推理正在快速去中心化，不再由闭源模型独占。

应用层面，Claude Artifacts 迎来重大升级。Cat Wu 宣布「Artifacts just got an upgrade」，Thariq 补充说新版本让 Artifacts 更具表现力，可以创意性地组合使用——比如在 Claude 中创建项目仪表盘。Nikunj Kothari 则展示了用 Ramp CLI + Claude Fable 一键构建费用管理自动化工具的全过程，甚至是在开车途中用语音 prompt 一次性完成的。Ryo Lu 更是用 Cursor 构建了一套自定义电子阅读器固件，涵盖拉丁文和 CJK 排版、竖排模式等，展示了 AI 辅助硬件黑客的无限可能。

本期播客

NVIDIA 如何构建开放前沿模型：Nemotron 负责人 Bryan Catanzaro 深度对话

The MAD Podcast with Matt Turck · YouTube

Matt Turck 采访了 NVIDIA Nemotron 负责人 Bryan Catanzaro，深入探讨 NVIDIA 的开放模型战略。Catanzaro 开篇即抛出核心洞察：「如果你接受我们已经处于极限的事实，那获得更多智能的方式就是提高效率。」他将 AI 工具比作人类的「外部器官」——厨房是外置的胃，而现在我们正在创造外置的大脑，其影响「深远到没人真正知道」。

对话深入 Nemotron 的技术细节：4-bit 训练、混合记忆 Transformer 架构、混合专家（MoE）、多 token 预测和多教师蒸馏。Nemotron 3 Ultra 发布时立即成为美国排名第一的开放权重模型，NVIDIA 在此方向上投入了数百名 AI 研究员。Catanzaro 还分享了现代 AI 研究组织的管理哲学——如何让众多天才研究员合力构建一个模型，而不是产出 100 篇互不相关的论文。

这期对话的核心启示是：NVIDIA 正在用「效率优先」的哲学重新定义前沿模型的构建方式。当算力瓶颈倒逼创新，开放模型不仅没有落后，反而在架构设计上展现出独特的工程优势。

用 Claude Foundation Models 框架构建 Apple 平台智能应用

Claude Blog · 2026-06-08

Anthropic 发布了新的 Swift 包，让 Apple 开发者可以通过 Foundation Models 框架调用 Claude。开发者只需三行代码即可在 Swift 中获得类型化返回值，结合 Apple 设备端模型和 Claude 云端能力，构建更复杂的智能工作流。

这篇文章标志着 Claude 正式进入 Apple 生态系统的核心开发链路。Foundation Models 框架为 Apple 开发者提供了原生的模型调用能力，而 Claude 的加入让这些应用可以处理远超设备端模型能力的复杂任务。对于 iOS 和 macOS 开发者来说，这意味着「AI 功能」不再是第三方 SDK 的专属，而是平台级的一等公民。

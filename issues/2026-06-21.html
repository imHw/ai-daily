Agent 时代的基础设施重构

当 Anthropic 的工程师们日常放手让 Claude 访问内部服务,当 Guillermo Rauch 宣称 Markdown 成为「下一个热门编程语言」,当 Aaron Levie 指出文件系统正成为人机协作的完美界面——这些看似分散的现象背后,实则是同一场基础设施革命:软件世界正在为 Agent 重新铺设轨道。Agent 不再是实验室里的演示品,而是倒逼整个技术栈回归本质的生产力工具。

Anthropic 的最新工程博客透露了一个关键转折:一年前他们绝不会给 Claude 足够权限去影响内部服务,如今这已是常态。风险管理的逻辑变了——不再是消除风险,而是控制「爆炸半径」。与此同时,开发者社区正用脚投票:Peter Yang 从 Claude Code 转向 Codex,核心原因是后者的浏览器和计算机控制能力让他「不用再满世界找 API」。工具的边界在消解,Agent 需要的是上下文和工作空间,而非更多接口。

Guillermo 的洞察一针见血:Agent 正在激励软件行业回归健康习惯——开放 API、完善文档、编写测试、拥抱 CLI、支持多格式。这不是技术的倒退,而是万维网最初愿景的回归。当 Intel 新 CEO Lip Bu Tan 在 66 岁接下「行业最难的工作」,当建造者们把 Agent 当作日常协作伙伴,我们看到的是务实主义的胜利:不是追逐概念,而是让工具真正改变生产方式。

Intel CEO 谈重塑半导体供应链

No Priors · 2026年6月

66 岁的 Lip Bu Tan 解释了为何接下 Intel CEO 这份「行业最难的工作」:这家标志性公司对半导体生态系统和美国太重要了。他分享了一个投资人的务实哲学——他投资的十家公司里有九家中途改过商业计划,因为市场在变。他相信团队而非个人英雄,强调要先谦卑地听客户的,强化财务,简化产品线,再驱动下一代领导性产品。

谈到当下,Lip Bu 对 Agentic AI 带来的 CPU 需求激增感到欣慰。更让他高兴的是老友黄仁勋投入的 50 亿美元支持如今已升值到 250 亿。他被问到的最意外经历?特朗普总统某天清晨要求他因利益冲突辞职,没有例外。他说服自己:我不需要这份工作,我做这个纯粹是为了拯救 Intel。

官方博客

Anthropic 如何在产品中容纳 Claude

anthropic.com

一年前 Anthropic 绝不会让 Claude 拥有足够权限去影响内部服务,如今这已是日常操作,开发者的生产力也因此提升。部署风险包含两个维度:故障概率和潜在损害。安全防护和模型训练的进步持续降低前者,而理论上的「爆炸半径」只会随着能力和访问权限扩展而增长。但当 Agent 能完成曾需要一个人甚至一个团队才能做的工作时,不部署的成本变得巨大,只要产品能做到安全,风险收益的天平就会大幅倾向采用。

工程问题变成了如何限制爆炸半径。当能对自主 Agent 的相对损害设定边界(比如通过控制其环境)时,高效用能力就能推动部署。Claude Mythos Preview 是一个例子——2026 年 4 月因爆炸半径过大未发布的模型。但随着防御方加固关键系统、安全防护成熟,类似能力级别的模型更广泛发布会变得合适,尽管总会存在一些风险。

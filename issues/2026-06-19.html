Codex 双重重置与 GitHub COO 谈 AI 编程未来

OpenAI 的 Thibault Sottiaux 宣布 Codex 迎来「双重重置」：不仅重置当前使用量，还额外赠送一个重置银行供用户自由使用。这预示着 AI 编程工具正在从严格的速率限制转向更灵活的使用模式。

GitHub COO Kyle Daigle 在最新访谈中深入探讨了 AI 编程代理的前线态势：GitHub 每月已处理 1700 万次代码合并请求，且呈指数增长。他同时揭示了开源维护者面临的新挑战——如何防止每月 200 美元的 AI 订阅费暴涨至 2000 美元的使用限额。

GitHub COO 解密：AI 为何尚未取代开发者

AI & I by Every · 6月17日

故事要从 GitHub 每月 1700 万次代码合并请求说起——这个数字正在指数级增长，让 GitHub 站在了 AI 编程革命的最前沿。

Kyle Daigle 揭示了一个反直觉的现象：尽管 AI 编程代理能力越来越强，但开发者角色并未被取代，反而变得更加关键。他透露团队正在开发一个「重置银行」系统，帮助用户控制 AI 工具的成本，避免订阅费失控。

访谈中最有趣的细节是：主持人 Mike Taylor 坦言自己用 AI 克隆了 Kyle 进行采访练习，而 Kyle 也透露了一个令人惊讶的反转——这种 AI 模拟人类互动的能力正在改变整个行业。

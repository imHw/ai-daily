Sol 增长引爆算力危机；Anthropic 揭示平台三层架构；AI 教育赛道开战

OpenAI 的 GPT-5.6 Sol 正在经历一场「疯狂增长」。Sam Altman 亲自发文感叹：「推理团队做了英雄般的工作来支撑需求」，但同时警告「可能会有些颠簸」。Thibault Sottiaux 更是直接抛出数据——ChatGPT Work 和 Codex 即将突破 900 万用户，并发起投票询问是否应该重置用量上限。与此同时，OpenAI 推出了 $100 Codex 积分活动，邀请用户分享对 GPT-5.6 Sol 的使用体验。这一系列动作表明：Sol 已经成为 OpenAI 历史上增长最快的产品，算力供给正面临前所未有的压力。

Anthropic 的平台战略在最新一期 Training Data 播客中首次完整曝光。平台负责人 Katelyn Lesse 和 Angela Jiang 揭示了一个三层抽象架构：底层是执行层（Execution），中间是知识层（Knowledge），顶层是即将重点投入的协调层（Coordination）。Angela 解释道：「如果 token 并非完全可互换——比如这个 token 在做建议、那个 token 在做执行——你就需要组合这些编排策略。」这与 Swyx 此前公开的多模型编排工作流不谋而合，暗示整个行业正在从「单一模型」向「策略编排」范式迁移。

应用层面，Anthropic 官方宣布 Claude for Teachers 正式上线，面向 K-12 教育场景。该产品从州级课程标准出发，通过 Learning Commons 连接高质量课程资源，自动生成教案和学生材料。隐私方面严格遵守 FERPA 合规要求，承诺不使用对话数据训练模型。Aaron Levie 则从更宏观的视角指出：代码之所以特别适合 Agent，是因为它「可以快速测试」——而大多数其他工作领域缺乏这种即时反馈机制，「企业需要为工作流建立更好的 eval 体系」。

Anthropic 平台负责人：我们构建的是生态，不是围墙花园

Training Data · YouTube · 2026-07-14

Training Data 主持人 Lauren 和 Patrick 采访了 Anthropic 平台负责人 Katelyn Lesse 和 Angela Jiang，深入拆解了 Anthropic 的平台战略。Angela 开篇即抛出核心框架：「在执行层和知识层之上，还有一个协调层。」她将协调层比作「元 harness」——底层 harness 为执行而设计，而上层则解决「如何让不同 token 承担不同角色」的问题。

Angela 透露了 Anthropic 的演进路线：「你会看到我们从执行层逐步向协调层迁移，推出越来越多的抽象工具。」Katelyn 则强调了 Anthropic 的生态哲学——「我们构建的是生态系统，而不是围墙花园」，这意味着 Claude 的能力将通过平台层开放给整个开发者社区，而非封闭在自家产品中。

这期对话的核心启示是：AI 平台竞争正在从「谁的模型更强」转向「谁的编排层更灵活」。当 token 变得不再是通用燃料而是需要分配角色的「专家」，平台的价值就体现在如何让开发者高效地组合这些专家——这正是 Anthropic 下一阶段的战略重心。

Claude for Teachers：面向 K-12 教育的 AI 助手正式上线

Claude Blog · 2026-07-14

Anthropic 发布 Claude for Teachers，正式进军 K-12 教育市场。该产品通过 Learning Commons 连接各州课程标准和高质量课程资源，教师只需描述需求，Claude 即可从标准出发生成完整教案和学生材料，供教师修改后带入课堂。

隐私保护是这款产品的核心卖点。Anthropic 承诺绝不使用教师对话数据训练模型，学生信息受到专门的数据处理协议保护，完全符合美国 FERPA（家庭教育权利和隐私法案）要求。这标志着 Anthropic 在教育领域的合规策略——用隐私合规作为差异化竞争优势，而非仅仅把 AI 塞进教室。
